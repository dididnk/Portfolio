Installation de Windex Server en tâche de fond

Ce chapitre explique, suivant la plateforme, le processus d'installation de Windex Server en tant que tâche de fond.

Ces procédures ne s'applique que dans le cas d'une installation de Windex Server avec son micro serveur propriétaire. Si vous avez intégré Windex Server à un serveur de servlet Tomcat, c'est le serveur Tomcat qui doit être lancé en tant que tâche de fond. Reportez-vous alors au site de Tomcat pour plus de détails.

1. Installation du service WindexServer sous Windows NT, 2000 ou XP

Si l'ordinateur sur lequel vous avez installé Windex Server fonctionne sous Windows NT, 2000 ou XP, le kit d'installation vous aura proposé l'installation de Windex Server en tant que service. Cependant, si vous n'avez pas choisi initialement cette option, vous pouvez l'activer à postériori.

Dans le dossier d'installation de Windex Server, dans le répertoire WEB-INF/bin, vous trouverez les fichiers suivants :

  • JavaService.exe
  • installService.bat
    Installe Windex Server en tant que service et démarre le service.
  • uninstallService.bat
    Arrête le service WindexServer et le désinstalle.
  • license.html
    Licence du programme JavaService.exe
  • JavaService-bin-1.2.0.zip
    Archive du programme JavaService avec sa documentation.

Ces fichiers ont été configuré en fonction des renseignements donnés lors de l'installation de Windex Server (dossier d'installation...). Il n'est donc pas nécessaire de les modifier pour les utiliser.

Pour installer le service

Double-cliquez sur installService.bat. Contrôlez ensuite dans la gestion des services, l'état du service WindexServer.

Cette étape n'est à réaliser qu'une seule fois ! A l'issu de cette installation, Windex Server se démarre et s'arrête via le composant "Service".

Le non respect de la procédure d'arrêt du micro serveur peut occasionner la corruption irrémédiable de la base de données de Windex Server.

Le fonctionnement du service peut être observé par l'intermédiaire de deux fichiers log situés dans le dossier WEB-INF : service_stdout.log et service_stderr.log.

2. Installation du démon WindexServer sous Unix

Pour les systèmes Unix, le lancement d'une application en tâche de fond (appelée démon) est plus délicate. Il n'existe pas de procédure standard, car la mise en oeuvre d'un démon dépend du type Unix. Reportez-vous à la documentation d'administration de votre système pour plus de détails.

Dans le dossier d'installation de Windex Server, dans le répertoire WEB-INF/bin, vous trouverez les fichiers suivants :

  • restart_windexserver
    Lance ou arrête le démon WindexServer.
  • check_windexserver
    Utilisé par restart_windexserver, il vérifie si le démon WindexServer est déjà lancé ou non.

La syntaxe de restart_windexserver est :

restart_windexserver [start|run|stop]

  • start
    démarre WindexServer en tâche de fond
  • run   
    démarre WindexServer en premier plan
  • stop  
    arrête WindexServer

check_windexserver est réservé à restart_windexserver. Cependant, vous pouvez l'utiliser pour vérifier l'état du démon :

check_windexserver [-q]

retourne 0 si Windex Server est démarré, sinon 1 (récupérable via la variable $?) . L'option -q permet de fonctionner en mode "silencieux" : aucun message n'est affiché à l'écran.

Le fonctionnement du démon peut être observé par l'intermédiaire du fichier deamon_windexserver.log situé dans le dossier WEB-INF.
