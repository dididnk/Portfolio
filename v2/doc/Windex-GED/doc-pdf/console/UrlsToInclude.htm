Restrictions sur les URLs indexer

 

Si cette liste est vide, tous les documents sont indexés.
Indiquez dans cette liste la ou les formes que doivent respecter les URLs des documents à indexer.
Il suffit qu'une URL vérifie une de ces règles pour que cette adresse soit traitée.

Restriction Signification
*/Documents/* N'indexe que les dossiers Documents et leurs sous-dossiers
*/info1/*
*/info2/*  		N'indexe que les dossiers info1 et info2 leurs sous-dossiers.
*.htm N'indexe que les fichiers d'extention htm .
*/info2/*.htm N'indexe que les d'extention htm, situé dans le dossierinfo2

Saisissez la valeur dans la zone puis cliquez sur le bouton pour ajouter l'élément à la liste.

Pour modifier une URL présente dans la liste, corrigez la valeur, puis cliquez sur le bouton

Pour supprimer une URL, cliquez sur le bouton

Pour supprimer toutes les URLs déjà saisies, cliquez sur le bouton .
Vous devrez confirmer la suppression avant qu'elle ne soit effective.

Lorsque vous utilisez des caractères génériques pour spécifier un dossier ou un fichier, les formes que vous indiquez s'appliquent sur l'intégralité du chemin de l'URL.

Prenons l'arborescence suivante comme exemple :

L'adresse de départ de l'indexation pourrait être :

    http://www.monsite.fr/documents/mes_fichiers/

La page page11.htm située dans le dossier info1 a pour URL :

    http://www.monsite.fr/documents/mes_fichiers/info1/page11.htm

Si nous ajoutons la règle de restriction */info1/*, nous constatons que l'URL du fichier page11.htm la vérifie.

Par contre, l'adresse de départ de l'indexation

    http://www.monsite.fr/documents/mes_fichiers/

ne vérifie pas la règle. Par conséquent, elle est exclue de l'indexation !
Comme cette URL est celle utilisée pour commencer l'indexation, l'indexation s'arrête immédiatement.
