Types de documents à indexer

Le type du document est indépendant de l'extension du fichier !
Windex Server se base sur le code MIME d'un fichier pour déterminer son type.
MIME signifie "Multi-purpose Internet Mail Extensions".
C'est une spécification décrivant les formats de messages multimédias sur l'Internet.
Elle permet entre autre, d'identifier le type du contenu d'un fichier.
Plusieurs fichiers peuvent avoir le même code MIME.
Ainsi, les fichiers avec l'extension asp ou nsfsont reconnus comme étant de la même famille que les fichiers HTML.

Après avoir modifié les types de documents, vous devez impérativement cliquer que le bouton

TypeDescription
*/*

Filtre générique

Ce filtre est utilisé pour les documents ayant un format différent des formats traités par Windex Server (ci-dessous) tel que les images, les vidéos ...
Il n'indexe pas le contenu des documents mais il permet d'indexer leur nom, leur taille, leur type ...

Attora Archive

Archive contenant des documents Attora

Documents Attora

Excel
Excel(vnd)

Documents Microsoft Excel

Les fichiers au format Microsoft Excel peuvent être identifiés sous deux codes MIME différents, d'où la présence des deux cases à cocher. Dans le doute, nous vous conseillons de cocher les deux.

Windex Server indexe le texte,mais également le résultat des éventuelles formules contenues dans le document.

Les autres informations du document (telles que l'auteur, le sujet, le titre, les mots clefs...) sont traitées comme des balises META en HTML, sous leur nom usuel (author, subject, keywords...).

HTML

Documents HTML

Windex Server détecte tous les hyperliens du document HTML afin d'indexer les documents correspondants.

Il est très important que le jeu de caractères utilisé par vos documents HTML soit précisé. Cette information est indiquée généralement dans l'entête du document via la balise META HTTP-EQUIV :

<META HTTP-EQUIV="content-type" CONTENT="text/html; charset=ISO-8859-1">
Un jeu de caractères est une liste de caractères codés et reconnus par un système, défini par une norme. En France, on utilise principale le jeu de caractères ISO Latin-1 (également connu sous le nom "ISO 8859-1"). Ce jeu de caractères appartient à la famille de la norme ISO, qui définit des tables de 256 caractères :
  • Les 128 premiers caractères correspondent aux caractères standard ASCII.
  • Les 128 caractères suivants sont des signes diacritiques (caractères accentués, ligatures...)

La connaissance du jeu de caractères permet à Windex Server de déterminer les caractères formant un mot et donc d'indexer correctement un document.

La valeur de l'attribut ALT de la balise IMG est indéxée.

<IMG SRC="images/logo.gif" ALT="Ceci est le logo de notre société">

Exclusion d'une partie du document de l'indexation

Si vous souhaitez qu'une partie d'un document HTML ne soit pas indexée, il vous suffit de l'encadrer par les balises <WSNOINDEX> et <WSNOINDEX>.
Ces balises ne sont reconnues que par Windex Server et ignorées par les navigateurs ou outils HTML. Tout au plus, certains outils pourront vous signaler qu'elles sont inconnues.

<HTML>
<BODY>
Ce texte sera indexé par Windex Server<br>
<WSNOINDEX>
Par contre, celui-ci ne le sera pas.
</WSNOINDEX>
</BODY>
</HTML>

La balise <TITLE> est utilisée par Windex Server pour générer le titre du document présenté dans la page des résultats.

PDF

Documents Adobe Acrobat (PDF)

Le texte des documents PDF est indéxé, ainsi que les propriétés (titre, résumé ...).

Les PDF "images" (documents papiers numérisés) ne contiennent pas de texte à indexer.
Pour que Windex Server puisse les indexer, ces fichiers PDF doivent être "Searchable". En conséquence, ils doivent avoir été retraités par un logiciel d'OCR afin de convertir les images en texte dans le document PDF.

Pour qu'un fichier au format PDF soit traité il ne doit pas être protégé.
PowerPoint

Documents Microsoft PowerPoint

Pour les documents PowerPoint, seul le texte du document est indexé.
Les autres informations du document (telles que l'auteur, le sujet, le titre, les mots clefs...) sont traitées comme des balises META en HTML, sous leur nom usuel (author, subject, keywords...).

RTF

Documents Rich Text Format (RTF)

C'est un format de fichier textuel (norme ASCII ou ANSI) contenant des codes de formatage du document. Ces codes sont aussi exprimés en ASCII ou ANSI. On pourrait le comparer à un document HTML dans sa logique.

TEXT

Documents Text

C'est le format le plus simple : le texte est "plat", sans aucun enrichissement ni aucune mise en forme à part quelques tabulations.

Word

Documents Microsoft Word

Pour les documents Word, seul le texte du document est indexé. Les autres informations du document (telles que l'auteur, le sujet, le titre, les mots clefs...) sont traitées comme des balises META en HTML, sous leur nom usuel (author, subject, keywords...).
A noter que le titre du document Word n'est pas traité comme une balise META, mais comme le titre du document.
