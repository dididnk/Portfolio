URLs à indexer

Considérations générales sur les URLs

L'URL se divise en trois parties :

     protocole://hôte/chemin

  • le protocole : vous rencontrerez principalement les protocoles HTTP, HTTPS, et FILE.
  • l'hôte : cela peut être l'adresse Internet d'un ordinateur (195.5.210.10), d'un site (www.lug.com). A noter deux équivalences : l'adresse 127.0.0.1 et localhost qui tous les deux représentent l'ordinateur local.
  • le chemin : il indique la "localisation" de l'information. Sa syntaxe et son interprétation dépendent du protocole.

Quelques exemples :

    http://www.lug.com/index.htm
    file://localhost/C:/documents/mes_fichiers

Adresses de départ de l'indexation

Ce sont les adressses des différents sites à partir desquels l'indexation démarre.

Saisissez la valeur dans la zone puis cliquez sur le bouton situé à droite de la zone Nouvel élément pour ajouter l'élément à la liste.

Pour modifier une URL présente dans la liste, corrigez la valeur, puis cliquez sur le bouton situé en regard de la ligne.

Pour supprimer une URL, cliquez sur le bouton situé en regard de la ligne correspondante.

Pour supprimer toutes les URLs déjà saisies, cliquez sur le bouton . Vous devrez confirmer la suppression avant qu'elle ne soit effective.


Comment indiquer un fichier ou un répertoire ?
Il est possible d'indiquer des règles de restriction et d'exclusion sur les documents que Windex Server doit traiter. Ces règles s'appliquent systématiquement sur l'URL de chaque document.
Si vous indiquez un fichier (exemple : http://www.mon_site.fr/index.htm), le traitement effectue :
  • le contrôle du format de l'URL
  • l'analyse du contenu
  • l'extraction des hyperliens

Chaque hyperlien issu de l'extraction se voit appliquer le même traitement. La logique du traitement de Windex Server dans ce cas est identique à la démarche que vous suivriez si vous naviguiez dans tous vos documents en suivant chaque hyperlien.
Si un fichier est orphelin, c'est à dire qu'il n'est référencé par aucun autre fichier, il ne sera pas pris en compte. En effet, il n'y a dans ce cas aucun moyen d'atteindre ce document par un hyperlien et donc à plus forte raison, aucun moyen pour que Windex Server le trouve !

Le fonctionnement est similaire pour un répertoire. Ce dernier est en fait considéré comme un document: il est constitué d'hyperliens sur les fichiers et sous-répertoires qu'il contient ! Aucun fichier ne peut alors échapper au traitement de Windex Server.

L'image ci-dessus illustre l'affichage dans un navigateur Internet du répertoire images d'un site Web. Le répertoire est vu comme une page HTML.
Cette solution d'indexation est à privilégier lorsque :

  • vous n'avez pas de page(s) contenant d'hyperliens sur l'intégralité des documents
  • vous indexez des documents PDF liés entre eux (les liens entre documents PDF ne sont pas pris en compte par Windex Server).

Comme un répertoire est considéré comme une page HTML, son contenu est indexé au même titre qu'une page standard. Ainsi, dans l'exemple ci-dessus, si on recherche le mot "85012799", Windex Server proposera la page http://www.mysite.com/images ! Pour ne prendre en compte que les hyperliens, il faut déclarer les répertoires comme pages de type sommaire.


Si vous voulez indexer le dossier suivant :
    C:/mes_documents/
Vous devez écrire :
    file:///C:/mes_documents/index.htm
ou
    file:/C:/mes_documents/index.htm
ou
    file://localhost/C:/mes_documents/index.htm
Dans tous les cas, Windex Server uniformisera la syntaxe en :
    file://localhost/C:/mes_documents/index.htm

Beaucoup de serveurs Web sont configurés de telle sorte que si l'on indique un nom de dossier comme dernier élément de l'URL, une page par défaut est affichée : index.htm, index.html, default.htm, default.html, default.asp...
Si aucune page par défaut n'est disponible, c'est le contenu du dossier qui est affiché. Cependant, cette fonctionnalité, pour des raisons de sécurité, n'est pas toujours activée et l'accès au dossier peut être systématiquement refusé (erreur 403) si on n'indique pas le nom explicite d'une page.


Windex Server se limite aux hyperliens inclus dans l'hôte et le même chemin. Si vous indiquez comme adresse de départ de l'indexation http://www.mon_site.fr/index.htm, tous hyperliens vers un site extérieur au site http://www.mon_site.fr ne sera pas indexé.

Les hyperliens présents dans les fichiers PDF et Word ne sont pas pris en compte.

Il est important de noter que Windex Server indexe aussi le texte de l'attribut ALT de la balise HTML <IMG>.

<IMG src="image.gif" alt="Ceci est une image">

Le texte "Ceci est une image" sera indexé.

Nombre d'URLs autorisées par votre licence

Par défaut, Windex Server vous permet d'indexer au plus 1 site Web. Plusieurs adresses de départ de l'indexation peuvent correspondre au même site Web. Ainsi, les exemples suivants :

    http://www.monsite.fr/index.htm
    http://www.monsite.fr/france/index/htm
    http://www.monsite.fr/europe/index.htm
    http://www.monsite.fr/asia/

compteront pour un site.

Si vous atteignez le nombre maximum de site Web autorisé par votre licence, vous en serez informé par un message qui apparaîtra à la place de la zone d'ajout. Dans ce cas, vous ne pouvez que modifier ou supprimer les URLs existantes.
