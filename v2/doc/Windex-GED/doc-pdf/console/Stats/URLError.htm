Page des URLs en erreur

Si lors de la dernière indexation, certaines URLs n'ont pas pu être indexé, le bouton apparaît en bas de l'écran principal du tableau de bord.

Cette page contient la liste des URLs en erreur. Chaque URL est précédée du code erreur renvoyé par le serveur qui héberge celle-ci. Pour connaître la signification de ce code, il suffit de cliquer dessus.

En plus des codes d'erreur standards, les codes précédés de WS- indique des erreurs rencontrés par Windex Server lors de l'analyse de certains fichiers.

Le bouton vous permet de revenir à l'écran principal du tableau de bord.

Le bouton vous permet d'actualiser la page.

Le bouton vous permet d'éffacer la liste des erreurs.

Le bouton vous permet de supprimer les erreur de type HTTP (404, 505) de la liste des erreurs.

Le bouton vous permet de supprimer les erreur dut au filtre de Windex Server de la liste des erreurs.

Cette liste est réinitialisée après chaque indexation. Par conséquent, après voir identifié l'origine de l'erreur, et fait le nécessaire pour la corriger, elle ne sera prise en compte que lors de la prochaine indexation. Si toutefois elle réapparaît, c'est que la correction n'a pas été efficace.
